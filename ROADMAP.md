# MediTrack – Project Roadmap

## Phase 1: Problem Definition & Research (Completed)
- Domain selection: Healthcare
- Problem identification: Lack of personalized symptom tracking
- Literature review and gap analysis
- High-level system architecture design

## Phase 2: Prototype Design & Planning (Current – PBL-2)
- Define system components and data flow
- Design NLP-based symptom processing pipeline
- Plan ML-based risk prediction using historical data
- Create architecture diagrams and workflow
- Host live project presentation via GitHub Pages

## Phase 3: Initial Implementation (Future)
- Develop backend APIs for symptom input
- Implement basic NLP preprocessing
- Integrate a baseline ML model
- Store and retrieve patient history data

## Phase 4: Advanced Features & Evaluation
- Improve ML model performance
- Add trend analysis and alert mechanisms
- Build dashboards for insights visualization
- Testing and validation

## Phase 5: Final Deployment
- End-to-end system integration
- Documentation and final evaluation
